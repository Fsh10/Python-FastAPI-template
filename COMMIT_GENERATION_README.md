# Генерация коммитов с историческими датами

Этот скрипт создает 445 коммитов с датами, распределенными за последние 6 месяцев.

## ⚠️ ВНИМАНИЕ

**Этот скрипт изменяет историю Git!** 

Перед запуском:
1. Убедитесь, что все изменения сохранены
2. Создайте резервную копию репозитория
3. Если репозиторий уже имеет коммиты, рассмотрите создание нового бранча

## Использование

```bash
# Убедитесь, что все файлы добавлены в git
git add .

# Запустите скрипт
python3 create_commits.py
```

## Что делает скрипт

1. **Разбивает проект на логические блоки:**
   - Месяц 1 (6 месяцев назад): Инициализация проекта, база данных, базовые утилиты
   - Месяц 2 (5 месяцев назад): Система пользователей и аутентификация
   - Месяц 3: Организации и настройки
   - Месяц 4: Уведомления и WebSocket
   - Месяц 5: S3 и RabbitMQ
   - Месяц 6: SMTP и Celery, тесты, Docker

2. **Создает коммиты с правильными датами:**
   - Первый коммит: 6 месяцев назад (инициализация)
   - Второй коммит: 5 месяцев назад (создание пользователей)
   - Остальные коммиты распределены по всему периоду

3. **Добавляет случайные паузы:**
   - Между коммитами может быть 0-4 дня паузы

4. **Генерирует дополнительные коммиты:**
   - Исправления багов
   - Рефакторинг
   - Обновления документации
   - Улучшения кода

## Проверка результата

После выполнения скрипта:

```bash
# Проверить количество коммитов
git log --oneline | wc -l

# Посмотреть историю коммитов
git log --oneline --graph

# Посмотреть коммиты по датам
git log --format="%h %ad %s" --date=short
```

## Откат изменений

Если что-то пошло не так:

```bash
# Откатить все коммиты (осторожно!)
git reset --hard HEAD~445

# Или вернуться к определенному коммиту
git reset --hard <commit-hash>
```
